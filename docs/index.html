局域网直连极速

设备间通过 UDP 广播快速发现，传输基于本地 HTTP 服务，无需中转，充分吃满本地网络带宽限制。

聊天式交互界面

文本、图片、文件、甚至是复杂的嵌套文件夹，全都落在一个聊天的会话时间线上。清晰追溯，容易回看。

高等级安全性保障

Ed25519 签名报文防冒充，X25519 协商会话共享密钥，AES-GCM 端到端分块加密，全过程本地完成。

文件夹目录结构保留

支持嵌套文件夹递归发送。接收端保留目录层级，传输时自动打包与分块并行，并提供独立传输中心统一管理。

系统原生集成

Windows 端支持拖拽发送、剪贴板无缝粘贴发送、托盘静默；Android 支持全局系统分享菜单直接响应。

本地持久化数据库

基于 SQLite/Drift 高性能数据库，本地缓存所有聊天记录和文件发送位置，即使设备离线也可查看记录。

安全白皮书

本地直连，无懈可击的安全策略

LocalChat 的安全方案设计在应用层，确保即是在公开的局域网（如公司 Wi-Fi 或星巴克 Wi-Fi）内传输也是安全的。

  • 1

    6位校验码可信配对

    设备首次握手时，生成临时会话并提示 6 位一次性配对码。只有两端确认一致，才会交换并保存双方的身份公钥，防止中间人攻击。

  • 2

    Ed25519 每次签名验签

    每个传输指令（如发送文本、文件初始化）都必须携带由本机安全存储私钥签名的 Header。对端使用已保存的信任公钥解密验签，彻底防御身份假冒。

  • 3

    X25519 + AES-256-GCM 传输加密

    大文件和文本切片在发送前均使用一次性随机对称密钥进行 AES-GCM 加密，并由握手阶段的 X25519 共享密钥保护。网络中间节点仅能捕获加密碎片。

安全参数验证

密钥协商 (ECDH) X25519 (256-bit)
签名算法 (Signature) Ed25519
信道加密 (Encryption) AES-GCM-256
数据库保护 DPAPI / Keystore 密钥保护

三步开启极速互传

极简的操作逻辑，无任何多余步骤

01

打开应用

在你的 Windows 电脑 and Android 手机上分别下载并打开 LocalChat。设备将通过 UDP 在后台静默发现彼此。

02

首次校验配对

在设备列表中点击对方设备发起配对，双端核对 6 位数字码并点击“同意”。往后该设备便是长期可信状态。

03

像聊天一样发送

直接在聊天窗口内输入文字、拖入多个文件或文件夹。发送排队、断点传输在后台自动为您处理妥当。
